# Stage automatic Skill learning before activation

EasyWork may learn reusable procedures from successful work, corrections, and recovered failures, but background learning creates a source-owned Skill Candidate rather than mutating the active Skill library. A Skill Review must compare the proposal with existing Skills, remove transient or secret material, validate its package, and obtain user approval before activation; the reviewer is restricted to Skill read and staging capabilities and cannot perform unrelated side effects. Approved Skills have independent provenance and usage history, can be pinned or recoverably archived, and are never automatically deleted. This sacrifices immediate autonomous writes in exchange for auditability, safety, and protection against persisting accidental or environment-specific behavior.
