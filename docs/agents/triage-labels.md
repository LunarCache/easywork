# Triage Labels

The skills use five canonical triage roles. This repository also defines `done`
as a terminal state for completed local Markdown issues.

| Canonical role | Tracker label | Meaning |
| --- | --- | --- |
| `needs-triage` | `needs-triage` | Maintainer needs to evaluate the issue |
| `needs-info` | `needs-info` | Waiting on the reporter |
| `ready-for-agent` | `ready-for-agent` | Fully specified and ready for an AFK agent |
| `ready-for-human` | `ready-for-human` | Requires human implementation |
| `wontfix` | `wontfix` | Will not be actioned |
| Repository extension | `done` | Work has been completed |

When a skill mentions a canonical role, use its corresponding tracker label.
Use `done` when a local Markdown implementation issue has been completed.
